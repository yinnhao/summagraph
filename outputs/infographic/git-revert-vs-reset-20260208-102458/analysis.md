- 学习目标：读者将理解git revert的核心原理、与git reset的关键区别及适用场景
- 核心数据点：
  - git revert操作方式：增加一个新提交，抵消旧提交
  - git reset操作方式：移动HEAD指针，丢弃之后的提交
  - git revert历史记录：保留完整的历史，公开透明
  - git reset历史记录：抹除历史，看起来像没发生过
  - git revert安全性：高，不会丢失代码
  - git reset安全性：中/低，如果用了--hard且没备份，代码会丢失
  - git revert适用场景：已推送（Push）到远程的分支
  - git reset适用场景：仅限本地尚未推送的分支
- 视觉机会映射：
  - 提交历史变化：用流程图展示A→B→C→D的过程
  - 命令对比：用对比表格或左右分栏的图标+文字展示
  - 金律提示：用醒目的警告框或图标突出显示
